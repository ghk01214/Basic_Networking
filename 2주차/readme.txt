구현 방법

노드 A 부분(발신부)
while문을 돌렸을 때 받은 node_id가 A 또는 a일 때 노드 A 진입한다.
노드 A에 진입하면 전송할 노드의 ID와 문자열을 입력받는다.
입력이 끝나면 노드로 전송을 시작한다는 의미의 true값을 전송한다.

전송이 시작되면 처음으로 보낼 노드의 정보(b~d)를 전송한다.
정보를 보내면 0.2초 쉰다.
입력한 문자열의 첫 문자를 제외한 문자열의 길이를 전송한다.
정보를 보내면 0.2초 쉰다.
마지막으로 문자열의 첫 문자의 다음 부분부터 문자를 전송한다.
문자를 1개 전송할 때마다 0.2초 쉰다.

노드 B~D 부분(수신부)
while문을 돌렸을 때 받은 node_id가 B~D일 때 노드 입력된 노드로 진입한다.
노드 B~D에 진입하면 A 노드와의 연결을 기다린다.
노드 A와 연결되고 문자열이 전송되기 시작하면 싱크를 맞추기 위하 0.01초 쉰다.
이후 발신된 노드 정보를 판독해서 진입한 노드 ID와 수신받은 노드 ID가 다르면 g_conn의 값을 false로 바꾸고 while문의 처음으로 돌아간다.
g_conn의 값을 false로 변경하는 이유는 추가적인 작업의 방지를 위해서다.
이후 같은 노드라고 확인되면 "노드 A의 메세지를 수신 중"이라는 문구가 뜨며 작업을 이어나간다.

작업이 재개되면 우선 출력할 문자열의 길이의 정보를 받아온다.
정보를 받으면 0.2초 쉰다.
다음으로 string 변수를 선언하고 출력할 문자열의 길이만큼 resize 함수를 사용하여 resize한다.
이후 문자를 보낸 순서대로 받으며 다 받으면 0.2초 쉬고 다음 문자를 받는다.
전부 다 받으면 받은 문자열을 출력한다.
문자열을 출력하면 추가 작업 방지를 위해 g_conn의 값을 false로 바꾼다.
